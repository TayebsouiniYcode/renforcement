# 🟢 LUNDI — Programme de Sauvetage (2h)

> **Objectif** : Combler les dernières lacunes et préparer le discours de soutenance  
> **Niveau** : Tous niveaux

---

## S-Challenge 1 — Auto-diagnostic

Note-toi de 1 à 5 sur chaque notion : tableaux JS, objets JS, async/await, Fetch API, SELECT/WHERE SQL, JOIN SQL, classes PHP, héritage PHP, encapsulation, PDO, UML cas d'utilisation, UML classes, Git. Pour chaque note ≤ 2 : écris ce qui te bloque.

---

## S-Challenge 2 — Révision ciblée (30 min)

Choisis tes 2 notes les plus basses de l'auto-diagnostic. Fais un exercice rapide sur chacune :
- Si SQL : écris 5 jointures sur la BDD école
- Si PHP : crée une classe avec héritage et getters/setters
- Si JS : écris 3 fonctions avec map/filter/reduce
- Si UML : dessine un diagramme de classes pour une boutique en ligne (3 classes)

---

## S-Challenge 3 — Simulation orale (3 min)

Prépare un discours de 3 minutes qui explique ton projet fil rouge : qu'est-ce que c'est (2 phrases), quelles technologies (liste), comment c'est organisé (structure), la fonctionnalité dont tu es le plus fier, ta plus grande difficulté. Chronomètre-toi. Enregistre-toi et réécoute.

---

## S-Challenge 4 — Les questions du jury

Écris une réponse courte pour : "C'est quoi MVC ?", "Pourquoi password_hash ?", "C'est quoi une injection SQL ?", "Explique une de tes jointures", "Comment marche l'authentification dans ton app ?", "Qu'est-ce que tu améliorerais ?". Entraîne-toi à l'oral.

# 🔴 MARDI — Renforcement : Révision conception et back-end (1h)

---

## R-Challenge 1 — Quiz technique écrit (20 min)

Réponds sans chercher sur Google :

**JavaScript (5 questions)** :
1. Quelle est la différence entre `map()` et `forEach()` ?
2. Que retourne `filter()` ?
3. Écris une Promise qui se résout après 1 seconde avec "Hello"
4. Quelle est la différence entre `.then()` et `await` ?
5. Comment faire une requête POST avec Fetch ? (les 3 éléments clés)

**PHP POO (5 questions)** :
1. Quelle est la différence entre `private` et `protected` ?
2. C'est quoi une classe abstraite ? Donne un exemple d'utilisation.
3. C'est quoi une interface ? Quelle différence avec une classe abstraite ?
4. Explique le pattern Singleton en 3 phrases.
5. C'est quoi le polymorphisme ? Donne un exemple.

---

## R-Challenge 2 — Quiz SQL et UML (20 min)

**SQL (5 questions)** :
1. Écris un INNER JOIN entre `produits` et `categories`
2. Quelle est la différence entre INNER JOIN et LEFT JOIN ?
3. Écris une requête avec GROUP BY et HAVING
4. C'est quoi une procédure stockée ? À quoi ça sert ?
5. C'est quoi une vue SQL ? Donne un exemple

**UML (5 questions)** :
1. C'est quoi un acteur dans un diagramme de cas d'utilisation ?
2. Quelle est la différence entre `<<include>>` et `<<extend>>` ?
3. Que signifie la cardinalité `0..*` ?
4. C'est quoi la composition (losange plein) en UML ?
5. Comment transforme-t-on une relation `*..* ` en BDD ?

---

## R-Challenge 3 — Mise en situation individuelle (20 min)

Tire une de ces questions au hasard et réponds au tableau ou à l'oral devant le groupe :
- Dessine le diagramme de classes d'une application de covoiturage (3 classes, relations, cardinalités)
- Écris au tableau une jointure SQL sur 3 tables et explique-la
- Explique la différence entre héritage et composition en POO avec un exemple concret
- Explique comment tu sécurises une application web PHP (3 mesures minimum)
- Dessine l'architecture client-serveur d'une application web et explique le flux

# 🔴 MERCREDI — Renforcement : Révision front-end et full-stack (1h)

---

## R-Challenge 4 — Expliquer son architecture

Prépare une explication de 5 minutes de ton projet fil rouge couvrant : l'architecture technique (quels dossiers, quel rôle), les choix technologiques (pourquoi PHP natif ou Laravel, pourquoi MySQL, pourquoi Fetch API), le schéma de la base de données (combien de tables, quelles relations), le flux d'une fonctionnalité principale (de A à Z : l'utilisateur clique → le front envoie une requête → le back traite → la BDD répond → le front affiche).

---

## R-Challenge 5 — Code review croisée

Échange ton code avec un autre apprenant. En 15 minutes, lis son code et note :
- 3 choses bien faites (ce que tu comprends facilement, ce qui est propre)
- 3 points à améliorer (variable mal nommée, code dupliqué, manque de commentaire, faille de sécurité)
- 1 question que tu te poses sur son code

Donne-lui ton feedback à l'oral en 5 minutes. Écoute son feedback sur ton code.

---

## R-Challenge 6 — Amélioration du dossier projet

Vérifie que ton dossier projet contient : la description du projet (contexte, besoin, objectifs), les diagrammes UML (exportés en image), le schéma de la base de données, la description de l'architecture technique, la liste des fonctionnalités réalisées, les difficultés rencontrées et comment tu les as résolues, les améliorations possibles si tu avais plus de temps. Si une section manque, écris-la maintenant.

# 🔴 JEUDI — Renforcement : Simulation de soutenance (1h)

---

## R-Challenge 7 — Simulation complète (20 min par apprenant)

Passe devant le formateur comme si c'était le vrai jury :

**Partie 1 — Présentation du dossier (5 min)** : Montre tes diagrammes UML. Explique le contexte et le besoin. Décris l'architecture technique.

**Partie 2 — Démonstration technique (10 min)** : Ouvre ton application et montre le parcours complet. Montre 2-3 extraits de code que tu maîtrises bien. Montre la BDD dans MySQL Workbench.

**Partie 3 — Questions (5 min)** : Le formateur pose des questions techniques. Réponds calmement. Si tu ne sais pas : dis "Je ne suis pas sûr mais je pense que..." plutôt que rien.

---

## R-Challenge 8 — Plan d'action post-simulation

Après ta simulation, le formateur te donne un feedback. Écris :
- Mes 3 points forts identifiés

- Les actions concrètes que je vais faire demain (vendredi) pour m'améliorer
- Les 2 questions du formateur auxquelles j'ai mal répondu → prépare de meilleures réponses

# 🏆 VENDREDI — Évaluation Finale et Bilan (2-3h)

---

## Format de la journée

**Matin — 2ème simulation de soutenance améliorée**

Chaque apprenant repasse avec les améliorations intégrées depuis le feedback de jeudi. Le formateur remplit la grille de compétences finale. Cette simulation est la dernière répétition avant le vrai jour J.

---

**Après-midi — Bilan individuel avec le formateur**

Entretien individuel de 15 minutes avec chaque apprenant :

1. **Points forts** : ce que tu maîtrises bien, ce dont tu peux être fier
2. **Axes d'amélioration** : ce qu'il faut encore travailler avant le fil rouge
3. **Objectifs concrets** : 3 objectifs mesurables pour le fil rouge
4. **Ressources recommandées** : liens et tutoriels personnalisés selon tes lacunes
5. **Encouragement** : rappel de tout ce que tu as accompli en 7 semaines

Chaque apprenant repart avec un bilan écrit et signé par le formateur.

---

## Grille de compétences évaluées

L'évaluation finale porte sur l'ensemble du programme :

- **JavaScript** : manipulation de données, logique métier, async/await, Fetch API, DOM
- **PHP POO** : classes, héritage, polymorphisme, interfaces, static, final, MVC, PDO
- **SQL/T-SQL** : requêtes, jointures, procédures stockées, fonctions, vues, index
- **UML** : diagrammes de cas d'utilisation et de classes
- **Sécurité** : hash des mots de passe, requêtes préparées, XSS, validation
- **Git** : versionning, commits, branches, GitHub
- **Communication** : expression orale, présentation technique, réponses aux questions

---

## 🎯 Fin du programme

Ce n'est pas la fin de ton apprentissage, c'est le début de ton projet fil rouge. Tu as maintenant les outils et les compétences pour le réussir. Bonne chance !
